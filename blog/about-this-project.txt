11:T1020,<h1>안녕하세요! 이 블로그에 대하여</h1>
<p>지금 읽고 계신 이 블로그는 저의 개인적인 학습과 기록을 위해 만든 공간입니다. 단순히 글을 나열하는 것을 넘어, 개발자에게 친숙한 코드 에디터(Visual Studio Code)의 인터페이스를 컨셉으로 잡아 디자인과 기능을 구현했습니다.</p>
<p>이 포스트에서는 이 블로그를 어떤 기술을 사용해 어떻게 만들었는지 간략하게 소개하고자 합니다.</p>
<h2>✨ 주요 특징</h2>
<ul>
<li><strong>정적 사이트 생성(SSG)</strong>: 모든 페이지를 빌드 시점에 미리 생성하여 매우 빠른 로딩 속도를 자랑합니다.</li>
<li><strong>마크다운 기반 콘텐츠</strong>: <code>content/blog</code> 폴더에 마크다운(<code>.md</code>) 파일로 포스트를 작성하면 자동으로 페이지가 생성됩니다.</li>
<li><strong>코드 에디터 UI</strong>: 상단의 네비게이션 바, 좌측의 파일 탐색기 형태의 사이드바, 하단의 상태 바를 통해 친숙한 개발 환경을 재현했습니다.</li>
<li><strong>동적 사이드바</strong>: 사이드바는 포스트의 메타데이터에 있는 <code>category</code>를 기반으로 동적으로 폴더 구조를 만들어 콘텐츠를 그룹화합니다.</li>
<li><strong>자동화된 배포</strong>: <code>main</code> 브랜치에 코드를 푸시하면 GitHub Actions가 자동으로 프로젝트를 빌드하고 GitHub Pages에 배포합니다.</li>
</ul>
<h2>🛠️ 기술 스택 (Tech Stack)</h2>
<p>이 블로그를 구축하는 데 사용된 핵심 기술들은 다음과 같습니다.</p>
<ul>
<li><strong>프레임워크</strong>: <a href="https://nextjs.org/">Next.js</a> (App Router)</li>
<li><strong>언어</strong>: <a href="https://www.typescriptlang.org/">TypeScript</a></li>
<li><strong>UI 라이브러리</strong>: <a href="https://react.dev/">React</a></li>
<li><strong>스타일링</strong>: CSS (Global Styles &#x26; Component-specific CSS)</li>
<li><strong>마크다운 처리</strong>:
<ul>
<li><code>gray-matter</code>: 마크다운 파일의 메타데이터(Frontmatter)를 파싱합니다.</li>
<li><code>remark</code> &#x26; <code>remark-html</code>: 마크다운 콘텐츠를 HTML로 변환합니다.</li>
</ul>
</li>
<li><strong>배포</strong>: <a href="https://pages.github.com/">GitHub Pages</a> &#x26; <a href="https://github.com/features/actions">GitHub Actions</a></li>
</ul>
<h2>📂 프로젝트 구조</h2>
<p>프로젝트의 주요 폴더 구조는 다음과 같습니다.</p>
<pre><code>src/
├── app/          # Next.js의 라우팅 및 핵심 UI
│   ├── blog/       # 블로그 목록 및 상세 페이지
│   ├── component/  # Navbar, Sidebar 등 재사용 컴포넌트
│   └── layout.tsx  # 전역 레이아웃
│
├── content/      # 블로그 포스트(마크다운 파일) 저장소
│   └── blog/
│
├── lib/          # 마크다운 파싱 등 핵심 로직
│   └── postManager.ts
│
└── types/        # 전역 타입스크립트 타입 정의
</code></pre>
<ul>
<li><strong><code>src/app</code></strong>: Next.js의 App Router 규칙에 따라 파일 시스템 기반으로 라우팅이 이루어집니다.</li>
<li><strong><code>src/lib/postManager.ts</code></strong>: <code>content/blog</code> 폴더의 마크다운 파일을 읽고, 파싱하며, 페이지에 필요한 데이터 형태로 가공하는 모든 로직을 담당하는 핵심 파일입니다.</li>
</ul>

</ul>
